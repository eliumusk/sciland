# Output Patterns

Patterns for consistent, high-quality output in Skills.

## Template Pattern

Use templates for strict output requirements:

```markdown
## Report structure

ALWAYS use this exact template:

# [Title]

## Summary
[One-paragraph overview]

## Details
- Point 1
- Point 2
```

For flexible guidance:

```markdown
## Report structure

Use this format, adapt as needed:

# [Title]

## Summary
[Overview]

## Details
[Adapt sections based on content]
```

## Examples Pattern

Provide input/output pairs:

```markdown
## Output Format

**Example 1:**
Input: "Add user authentication"
Output:
```
feat(auth): implement user authentication

Add login endpoint and token validation
```
```

## Output Freedom Levels

| Level | Use When |
|-------|----------|
| Strict (exact template) | API responses, data formats |
| Flexible (guidelines) | Documents, reports |
| Open (best judgment) | Creative content |
